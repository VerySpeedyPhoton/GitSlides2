Pull Request / Review Demonstration!

  1. (On Github): Fork repository
  2. (In terminal): "git clone" the forked version of the repo
  3. (In terminal): Create a new branch, and switch to it.
  4. (In terminal): Edit code, "git add", "git commit"
  5. (In terminal): Push completed branch to Github: "git push -u origin [new-branch]"
  6. (On Github): The new branch should be visible. Now click 'New pull request'.
  7. (On Github): Submit the pull request (right side: "from" new-branch, left side: "to" main)
  8. (On Github): Maintainer logs in, sees the Open pull request, and reviews the changes.
  9. (On Github): (Complete additional code reviews / checks as needed.)
  10. (On Github): If pull request is accepted, the Maintainer merges it with 'Confirm merge'.
  11. Contribution is complete!
  12. And it works!!

Individual Workflow Demonstration (with Github Pages!)

Step 1: Set-up

  1. Install Node.js (10.0.0 or later)
    • (Only needed because we're using reveal.js)
    • This is for the "npm start" command for testing the presentation slides locally.
      •
  2. Make sure you've created a Github access key.
    • Only needs to be done if you've never remotely connected to your Github before.
    • Give sufficient permissions: check the boxes for 'repo' and 'workflow'
  3. Go to your Github repos and make a new, empty repository.
    • 'Your repositories' > 'New'
    • No need to change anything; don't add a README file, etc.

Step 2: Connect your local machine to your remote Github repository


					#Create new folder & enter it:
					mkdir demo && cd demo
					
					# Create placeholder file:
					echo "# placeholder" >> README.md

					# Initialize git inside the folder
					git init
					git add .
					git commit -m "first commit"
					git branch -M main	# won't work without a prior commit

					# Establish connection to remote repository.
					git remote add origin [link to your new empty repository]
					git push -u origin main  # Enter Github username, & access key

Step 3: Initialize reveal.js


					#We're going to play around with the reveal.js sample slidedeck.
					# Clone the 'reveal.js' repository:
					git clone https://github.com/hakimel/reveal.js.git

					# Move to the reveal.js folder and install dependencies:
					cd reveal.js && npm install	# install only needs to be done once.

					# View the slidedeck (hosted on local machine):
					npm start

					# To view the slides:
					# Open a webbrowser and go to `http://localhost:8000`.

					# Type 'Control-c' to exit the 'npm start' command.

Step 4: Modify the slides.



					#Create a new branch for our changes:
					git checkout -b newFeature

					# Edit the index.html file; add a new slide!
					vim index.html		# a normal text editor works too.
				
					# Add our changes to the staging area:
					git add index.hmtl

					# Commit our changes:
					git commit -m "added new slide"

					# Merge our newFeature branch with the main branch:
					git checkout main
					git merge newFeature

Step 5: Host the slidedeck as a website on Github!



					#Our new repository still only has a placeholder file.

					# Re-establish connection to remote Github repo:
					git remote remove origin 
					git remote add origin [link to the repository]
					# ^ Needed because 'git clone' changed our origin earlier.

					# Let's push our commits to the Github repo:
					git push -u origin main
		
					# Now, go to your Github repo.
					# 'Settings' > 'Pages' > 'Select branch: main' > Save
					# Wait a little while, refresh the page.
					# A clickable link should appear for your deployed site :)

Step 6: Make some more edits & update your deployed site.


					#Let's change the title slide.
				
					# Edit the index.html file
					# (demo.html has the code for the full reveal.js demo slides)
					git checkout newFeature
					vim index.html

					# Save newFeature changes to main:
					git add index.html
					git commit -m "Added Title"
					git checkout main
					git merge newFeature

					# Push commit to online repository:
					git push
